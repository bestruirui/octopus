# 首页分析概览整合设计

## 目标

让首页从分散的统计卡片页升级为更完整的总览页，把分析中心的“概述”指标并入首页，同时移除分析中心中的“概述”标签，避免重复信息入口。

## 设计范围

- 首页新增一个更完整的 Hero 区域，承载标题、副文案和 4 个高优先级信号。
- 首页新增“分析概览”区，复用 `/api/v1/analytics/overview` 的 8 个核心指标。
- 首页分析概览默认展示最近 `7天`，并提供轻量范围切换。
- 首页重排剩余模块顺序，让“总览 -> 分析概览 -> 趋势 -> 细节”更顺。
- 分析中心移除“概述”标签，仅保留“利用率 / 路由健康 / 评估”。

## 不做的事

- 不改后端接口或统计口径。
- 不新增首页独立 API。
- 不把分析中心其他模块搬到首页。
- 不重做现有趋势图、排行榜的数据逻辑。

## 信息结构

### 首页

1. Hero
2. 分析概览
3. 趋势图
4. 活跃热力图 + 排行榜双列区

### 分析中心

1. 利用率
2. 路由健康
3. 评估

## 视觉方向

- 保留当前浅色、圆角、柔和阴影的产品语言。
- Hero 采用更强的渐变背景和层次感，解决当前标题区过空的问题。
- 分析概览使用紧凑的 `2 x 4` 指标网格，和 Hero、趋势图形成清晰节奏。
- 热力图补上标题与说明，避免当前裸矩阵视觉断层。

## 实现边界

- 复用现有 `useAnalyticsOverview()`，由 React Query 做请求共享。
- 首页新增独立的 overview range 状态，默认 `7d`。
- 保留 `Overview.tsx` 作为可复用指标渲染组件，但不再在分析中心 tab 中暴露。

## 验证

- 前端类型检查与构建可通过。
- `test:i18n` 可通过，确保新增文案键三语一致。
- 浏览器中确认首页顺序、Hero 层次、分析概览、分析中心 tab 变化都符合预期。
