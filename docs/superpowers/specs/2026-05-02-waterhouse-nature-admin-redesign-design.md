# Waterhouse Nature 后台重设计规格

## 背景

当前 `octopus-dev` 前端已经有一层初始的 Nature 风格基础：

1. 全局有浅青绿色系、Blob 背景、粒子背景、涟漪轨迹和部分 gooey 滤镜
2. 通用组件如 `Card`、`Button`、`Input` 已经开始引入磨砂玻璃、柔阴影和有机圆角
3. 主界面骨架仍然是典型后台：
   - 固定导航 + 页头 + 工具栏
   - 页面主体主要靠规则网格和白卡堆叠
   - 表单、弹层、筛选、列表仍然偏办公后台语义

用户本次要求不是“补一点自然元素”，而是把整套后台系统重组为更完整的 Nature 风格，并明确选择了：

1. 偏激进
2. 视觉优先
3. 允许大改页面结构和导航组织
4. 主基调选择“水雾疗愈”

因此本次设计的目标不是局部润色，而是把后台从“普通管理面板 + 自然装饰层”升级为“水雾温室式的数字仿生界面”。

## 目标

1. 将现有后台统一重构为 `Waterhouse / 水雾温室` 风格。
2. 让 Nature 风格从背景装饰层深入到：
   - 导航结构
   - 页头与工具操作
   - 卡片与面板组织
   - 表单、筛选、弹层、列表、图表容器
   - hover / active / 过渡动画
3. 在保持管理后台可用性的前提下，大幅削弱传统 B 端的工业化栅格和硬边框观感。
4. 以首页、设置、渠道、分组、模型广场、分析中心为主战场，建立一整套可复用的 Nature 视觉和交互系统。

## 非目标

1. 不改后端接口、权限模型、统计口径或数据库结构。
2. 不新增路由，不重命名现有模块语义边界。
3. 不把页面改造成不可读的纯展示型艺术页面。
4. 不在本轮顺手修改无关业务逻辑。
5. 不引入完全独立的新设计系统；优先在现有 `web/src/components/ui/*` 和 `web/src/components/nature/*` 基础上推进。

## 总体设计方向

本次统一采用 `方案 A：水雾温室`。

核心语义如下：

1. 后台不是“仪表盘墙”，而是“漂浮在雾中的液态控制温室”
2. 导航不是按钮列，而是“导航脊柱”
3. 页头不是标题栏，而是“雾冠”
4. 页面主体不是整齐白卡，而是“多器官舱室”
5. 表单不是办公控件，而是“命令培养皿”

整体氛围关键词：

1. 晨雾
2. 浅水
3. 磨砂
4. 漂浮
5. 缓慢呼吸
6. 柔性边界

## 信息架构重组

### 全站壳层

全站壳层从当前的：

1. 左侧圆角导航
2. 顶部标题 + 工具栏
3. 主体内容区

升级为：

1. `导航脊柱`
   - 更厚、更圆、更像漂浮骨节
   - 当前激活项用液滴包裹态表达
2. `雾冠页头`
   - 标题、副说明、关键操作、快速状态汇总统一纳入一个漂浮页头区
3. `多器官舱室`
   - 每页主体由不同尺寸、不同曲率、不同重量感的舱室组成
4. `命令舱`
   - 高频操作、筛选、创建、编辑入口可以从零散工具栏转移到更明确的操作舱体

### 页面结构原则

1. 拒绝“每个模块都是同一张矩形卡片”
2. 允许不对称布局，但必须保持阅读主轴清晰
3. 不同业务区块通过：
   - 曲率
   - 透明度
   - 亮度
   - 阴影深浅
   - 舱体大小
   来区分层级，而不是只靠细灰线

## 视觉系统

### 色彩

主基调采用浅雾青绿和冷水蓝绿，不走深色森林路线，也不走强实验液态霓虹路线。

色彩角色：

1. 主背景：雾白 + 浅灰绿
2. 主强调：浅青绿
3. 次强调：雾蓝
4. 辅助高光：近白色水面反光
5. 禁止回退到通用后台蓝灰配色逻辑

### 材质

全站统一材质规则：

1. 磨砂玻璃为默认容器材质
2. 重要舱体带轻微高光层
3. 表单和胶囊按钮使用更实的“液态实体”材质
4. 低对比柔阴影替代传统厚重投影

### 边界

边界规则：

1. 统一削弱直线和硬矩形
2. 普通容器允许大圆角
3. 关键容器使用不完全对称曲率
4. 页面中不同卡片不能全部共享同一个圆角模板

## 交互与动画系统

### 永动层

即使无操作，界面也保持极弱动态：

1. 背景 Blob 漂移
2. 粒子缓慢漂浮
3. 关键舱体轻微呼吸

原则：

1. 只给生命感
2. 不制造焦虑和注意力噪音

### Hover

Hover 统一升级为：

1. 磁吸式轻微位移或拉伸
2. 柔光增强
3. 局部液滴包裹

不再满足于简单变色或轻微投影变化。

### Active

点击反馈统一升级为：

1. 果冻式轻压缩
2. 弹性回弹
3. 重要动作附带浅水波扩散

### 页面过渡

页面切换和模块切换保留流体过渡语义：

1. 避免生硬切页
2. 淡入、微 blur、位移和形态流动配合使用
3. 不追求炫技级大动画

## 组件级规范

### 导航

1. 当前 `NavBar` 升级为“漂浮脊柱”
2. 图标容器更厚、更软、更立体
3. 激活态采用液滴包裹和 gooey 融合感
4. 未激活项保留弱存在感，避免完全工具化

### 页头与工具操作

1. 当前标题区升级为“雾冠页头”
2. 当前 `Toolbar` 逻辑保留，但视觉和编排重构
3. 搜索、筛选、布局切换、创建操作不再零散并排，而是重新编排进页头和命令舱

### Card / Panel

1. 通用 `Card` 不再只是统一圆角玻璃板
2. 需要支持至少几种不同舱体轮廓
3. 卡片内部层次更多靠空间、光感和局部色温，而不是二级边框

### 表单

1. `Input` / `Select` / `Switch` / `Popover` / `Dialog` 统一液态化
2. 输入框由“后台输入框”升级为“培养皿式输入容器”
3. 长文本框和 JSON 编辑区要有更深的液池感
4. Focus 状态要带柔和发光和轻水波反馈

### Button

1. 主按钮是液态胶囊
2. 次按钮是薄雾胶囊
3. Icon button 不再只是透明图标按钮，而是有明确实体感

### Dialog / Morphing Dialog

1. 对话框是本次视觉升级重点
2. 要从普通面板弹窗升级为“水舱弹层”
3. 允许大尺寸不规则轮廓
4. 保留当前 `MorphingDialog` 的交互基础，但视觉要显著升级

## 页面级设计

### 首页

定位：`生态仪表盘`

改造方向：

1. Hero 区像湖面或浮叶群
2. 高优先级统计卡像漂浮观察叶片
3. 趋势图和排行区改造成更完整的主从结构
4. 首页承担整套风格的最佳样板页职责

### 设置页

定位：`生长岛群`

改造方向：

1. 继续保留多配置域事实
2. 每个配置域成为独立生态岛
3. 当前两列瀑布式结构保留基础可读性，但视觉上更有呼吸与层次
4. 长表单办公感要被明显削弱

### 渠道页

定位：`液态陈列架`

改造方向：

1. 渠道卡更像带状态液面的容器
2. 搜索、筛选、排序、创建重编排
3. 列表 / 网格都要摆脱典型后台卡片墙

### 分组页

定位：`路由温室`

改造方向：

1. 分组卡、空状态、AI 路由入口都适合做更强的器官式布局
2. 创建 / 编辑分组弹层作为重点设计对象
3. 空状态不再只是提示盒，而是带引导性的操作场景

### 模型广场

定位：`雾中标本馆`

改造方向：

1. 模型卡更像展柜和标本铭牌
2. 标签、成功率、价格等信息做轻层级浮层处理
3. 排序和筛选组件更具观赏性，但不影响扫读

### 分析中心

定位：`潮汐观测站`

改造方向：

1. Tabs 和 range 切换器升级为液态浮标式切换器
2. 图表容器更空灵，但图线和标签清晰度不能下降
3. 分析页保留功能性，但显著降低后台模板感

## 实现边界

### 可复用优先

优先在以下层做统一升级：

1. `web/src/app/globals.css`
2. `web/src/components/ui/*`
3. `web/src/components/nature/*`
4. `web/src/components/app.tsx`
5. `web/src/components/modules/navbar/*`
6. `web/src/components/modules/toolbar/*`

这样后续页面模块只需在新语言上做局部适配，而不是页页单独造样式。

### 不改的技术边界

1. 不改 API 协议
2. 不改页面 route id
3. 不改业务数据流
4. 不改虚拟滚动和 React Query 的核心使用方式

## 实施优先级

建议按以下顺序落地：

1. 全局壳层
   - 全局背景
   - `AppContainer`
   - `NavBar`
   - 页头 / `Toolbar`
2. 通用组件层
   - `Card`
   - `Button`
   - `Input`
   - `Select`
   - `Dialog`
   - `Popover`
   - `Table`
3. 首页
4. 设置页
5. 渠道 / 分组
6. 模型广场
7. 分析中心
8. 其余页面统一补齐

## 验证要求

1. `pnpm lint` 可通过
2. `NEXT_PUBLIC_API_BASE_URL=\"http://127.0.0.1:8080\" pnpm build` 可通过
3. 浏览器中逐页确认：
   - 风格统一
   - 动效不过量
   - 表单和操作仍然易懂
   - 首页、设置、渠道、分组、模型广场、分析中心都达到目标语义

## 风险与取舍

1. 这是一次激进重构，视觉收益大，但需要严格控制不要把所有业务页都做成难用的艺术展示板
2. 用户明确选择了视觉优先，因此允许信息密度下降，但不能破坏核心录入、筛选、创建路径
3. Nature 风格不适合把所有复杂列表都完全液态抽象化，因此高密度信息页必须做“容器强风格 + 内容仍可扫读”的折中
